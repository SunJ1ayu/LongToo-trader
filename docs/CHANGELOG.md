#### 基础设施
- 回测系统

- 单元测试

### 🐛 Bug 修复 - 交易次数计数逻辑（2026-04-08）
**问题**: 每日交易次数限制（3次）把卖出也计入了，导致止损后无法开新仓  
**影响**: 连续止损时会卡住，无法执行策略  
**修复**: `daily_trade_count` 只统计买入，卖出/止损卖出不计入  
**教训**: 注释说"不适用于止损卖出"，但计数逻辑没区分买卖方向 —— **注释和代码必须双向验证**

### 🐛 Bug 修复 - 模拟模式数据隔离（2026-04-08）
**问题**: `--dry-run` 模拟交易会更新 `daily_trade_count`、记录日志、保存止损价，污染真实风控状态  
**影响**: 测试后实盘交易被错误地限制（如"今日交易次数已达上限"）  
**修复**: 所有状态更新操作（update_after_trade, log_trade, save_stop_loss 等）添加 `if not self.dry_run` 保护  
**教训**: **模拟和实盘必须完全隔离**，不能共用任何持久化状态

### 🐛 Bug 修复 - 模拟模式完整隔离（2026-04-08 补充）
**问题**: 发现更多模拟模式未隔离的调用：
- `add_adjustment_log` 记录调整日志
- `update_daily_pnl` 更新盈亏统计  
- `task_graph.save()` 持久化任务图（9处）
**修复**: 所有上述调用添加 `if not self.dry_run` 保护

### 🐛 Bug 修复 - 除零保护（2026-04-08）
**问题**: 仓位计算和换仓逻辑中有多个除法操作没有除零检查  
**位置**: 
- `calculate_position_size` (memory_manager.py:495)
- `_execute_rotation` 换仓股数计算 (trader.py:963, 990)
**修复**: 添加 `price <= 0` 和 `cash <= 0` 检查  
**教训**: **所有除法操作前必须验证除数有效性**
